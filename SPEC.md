# 纸片人男友 - 产品需求文档

> 版本：v2.0
> 日期：2026-06-23
> 状态：开发中

---

## 一、产品概述

**产品名称**：纸片人男友
**产品类型**：AI 虚拟恋人聊天产品
**一句话描述**：免费的、有人设的虚拟男友聊天产品，让用户感觉在跟一个有性格的真人聊天。

---

## 二、问题陈述

### 目标用户

- **主要人群**：普通上班族女性，寻求情感陪伴
- **次要人群**：对虚拟恋爱感兴趣，但不想付费的用户

### 用户痛点

| 痛点 | 描述 |
|------|------|
| 现有产品机械 | 市面上的 AI 聊天产品回复千篇一律，没有根据用户的性格、年龄、爱好进行针对性回复 |
| 付费门槛高 | 优质体验需要付费，用户想免费获得情感陪伴 |

### 解决思路

通过以下方式解决机械感：
1. 预设多个有鲜明人设的虚拟男友角色
2. 在聊天中逐步了解用户信息（性格、年龄、爱好）
3. 根据用户画像生成个性化回复
4. 结合语音 + 图片增强沉浸感

## 三、方案描述

### 3.1 用户角色选择

- 用户从预设角色列表中选择一个虚拟男友
- 每个角色有独特的：
  - **性格**：如阳光开朗、内敛温柔、傲娇毒舌等
  - **外貌**：通过头像图展示
  - **背景故事**：简短的人物设定

### 3.2 用户信息收集

- **方式**：边聊边了解，通过巧妙设计的选择题与用户沟通
- **收集内容**：
  - 性格倾向
  - 年龄段（20-25岁 / 26-30岁 / 30+）
  - 兴趣爱好（追剧 / 游戏 / 美食 / 旅游 等）
- **应用**：LLM 根据用户画像调整回复风格

### 3.3 聊天互动

- **界面**：微信风格的聊天界面
- **输入**：用户自由打字
- **回复**：
  - 文字消息
  - 自动生成语音消息（用户可播放）
- **特殊状态**：
  - 显示"对方正在输入..."状态
  - 消息发送中显示 loading

### 3.4 AI 自拍照片

- **触发条件**：根据聊天内容智能触发
  - 例如：用户提到"加班"，AI 发送一张办公室场景照
  - 用户提到"周末"，AI 发送一张休闲场景照
- **照片风格**：
  - 与虚拟男友人设一致
  - 发送场景照（非人脸自拍）
- **频率**：平均每聊 5 分钟触发 1 次

### 3.5 分享与邀请

- **截图分享**：支持用户将聊天记录截图分享给闺蜜
- **邀请机制**：支持"邀请闺蜜一起玩"功能
  - 生成邀请链接/邀请码
  - 追踪邀请效果

### 3.6 其他功能

- **角色切换**：用户可随时切换其他虚拟男友
- **存档导出**：支持保存精彩对话片段
- **外链支持**：AI 可发送外部链接（如公众号文章）

## 四、技术约束

### 4.1 技术栈

| 类别 | 技术选型 |
|------|----------|
| 前端框架 | Next.js |
| 开发语言 | TypeScript |
| 样式方案 | Tailwind CSS / 内联样式 |
| AI 能力 | 豆包系列（扣子 + 火山引擎）|

### 4.2 AI 能力集成

| 能力 | 具体产品 | 说明 |
|------|----------|------|
| LLM 对话 | 扣子 Coze | 用于生成文字回复 |
| TTS 语音 | 火山引擎语音合成 | 自动生成语音消息 |
| 图像生成 | 火山引擎即梦/画文 | 生成场景照片 |

### 4.3 其他约束

- 全程中文交互
- 内容安全：依赖扣子和火山引擎平台内置过滤
- **用户注册登录**：支持手机号 + 验证码登录，可选微信授权
- **数据持久化**：使用 PostgreSQL 数据库存储用户数据、聊天记录、用户画像
- 消息存储：使用 React state（当前会话）+ 数据库（历史记录）
- 支持多端同步：用户换设备登录可恢复聊天记录

### 4.4 性能要求

- LLM 响应 + TTS 生成 + 图片生成的总延迟控制在 15 秒内
- TTS 语音播放成功率 ≥ 95%
- 聊天界面流畅，无明显卡顿

## 九、详细设计方案

### 9.1 预设角色人设清单

根据乙女游戏最佳实践，设计 **5个角色**，覆盖主流用户偏好：

| 角色类型 | 职业 | 性格关键词 | 人设要点 |
|----------|------|------------|----------|
| 霸道总裁型 | 互联网公司CEO | 表面高冷、内心柔软、有脆弱感 | 有商业帝国但内心孤独，只对用户展现柔软一面 |
| 阳光少年型 | 赛车手 | 开朗直接、有点调皮、少年感 | 永远意气风发，但也有细腻温柔的一面 |
| 温柔医生型 | 心外科医生 | 沉稳体贴、善于倾听、治愈系 | 见过太多生死，更懂得珍惜当下 |
| 神秘艺术家型 | 自由画家 | 浪漫神秘、有点傲娇、艺术气息 | 看起来玩世不恭，内心其实很深情 |
| 青梅竹马型 | 特警 | 正义感强、守护型、有点笨拙 | 从小一起长大，默默守护但不善表达 |

**设计原则**：
- 每个角色要有"内心脆弱点"，不是完美舔狗
- 双向奔赴感，不是围着用户转的工具人
- 有自己的目标和行动线

### 9.2 用户画像收集题库

**收集方式**：边聊边问，每次只问1道，自然融入对话

#### 第一类：兴趣爱好（聊天开场引入）

```
"周末一般喜欢干嘛呀？"
A. 在家追剧/打游戏
B. 出门逛街/喝咖啡
C. 运动健身
D. 宅着什么都不干
```

#### 第二类：性格倾向（情绪对话中探测）

```
"工作中遇到烦心事，一般会怎么发泄？"
A. 找朋友吐槽
B. 一个人消化
C. 买买买！
D. 吃好吃的
```

#### 第三类：年龄/经历（深度对话中自然带出）

```
"话说你是上班族吧？工作几年啦？"
A. 刚毕业1-2年
B. 3-5年了
C. 5年以上，老油条了
```

**设计原则**：
- 每次只问1道，不要连续追问
- 选完后AI要有简短反馈（"原来你喜欢追剧呀！"）
- 收集3-5个维度即可，不要贪多

### 9.4 邀请闺蜜机制

#### 核心流程

```
[用户点击"邀请闺蜜"] → [生成精美邀请海报] → [分享到微信]
                                          ↓
                          [闺蜜扫码/点击链接] → [进入选择角色页]
                                          ↓
                              [用户收到通知："她来了！"]
```

#### 邀请入口

- 聊天界面顶部固定"邀请闺蜜"按钮
- 支持生成海报图片（带小程序/网页入口）
- 支持复制邀请链接

#### 追踪机制

- 为每个用户生成唯一邀请码（用户ID哈希）
- 闺蜜通过邀请链接进入时，自动绑定邀请关系
- 闺蜜完成角色选择后，通知邀请人

#### 激励设计

- **即时反馈**：闺蜜加入后立即通知（"她来了！"）
- **不设硬性奖励门槛**：靠话题性和分享欲驱动传播
- **制造话题性**：截图分享本身就有话题（"这个男的是谁？？？"）

### 9.5 存档导出格式

提供 **4种导出格式**，满足不同场景：

| 格式 | 适用场景 | 实现方式 |
|------|----------|----------|
| **精美卡片图** | 朋友圈/小红书分享 | 生成带边框、滤镜的聊天截图 |
| **原生长图** | 保存回忆 | 纯对话截图，无装饰 |
| **收藏集** | 精华片段汇总 | 用户勾选多条对话，打包导出 |
| **语音+文字** | 完整记录 | 导出 JSON（含音频URL） |

#### 精美卡片示例

```
┌─────────────────────────┐
│  🌙 今晚的月亮很好看    │
│                         │
│  [聊天对话内容]         │
│                         │
│  ┌─────────────────┐   │
│  │ ▶ [音频播放条]   │   │
│  └─────────────────┘   │
│                         │
│  [角色名] · 收藏于 6.23 │
└─────────────────────────┘
```
